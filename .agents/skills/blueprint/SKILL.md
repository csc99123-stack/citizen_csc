# Skill: Blueprint

## Purpose
Ensures architecture and business rules align perfectly with the overarching PRD.

## Guidelines
- Read against `PRD.md` frequently to double-check acceptance criteria prior to committing completion states.
- Follow a highly specific, organized folder structure (`/app`, `/components`, `/lib`, `/hooks`, `/schemas`, `/actions`).
- Keep code clean, DRY (Don't Repeat Yourself), and thoroughly commented if there is obscure business logic in place.
- Run tests and ensure deployment checks pass seamlessly.
